# Team Role Split — ISRO Team SIRIus

## Team Members

| Member | Main Role | Responsibility Area |
|---|---|---|
| Rockie | Backend Lead + Project Coordinator | Backend development, process management, integration, team coordination |
| Venkyy | Research Lead | Research, concept clarity, explanation, documentation support |
| Laksha | Frontend Developer | Frontend pages, UI components, styling, user experience |
| Mru | Frontend Developer | Frontend pages, animations, visual polish, result-viewing interface |

---

## 1. Rockie — Backend Lead and Project Coordinator

Rockie will handle most of the backend development and also manage the overall project flow.

### Main Responsibilities

- Build and manage the backend using FastAPI.
- Create API endpoints for image upload, processing status, results, and downloads.
- Connect the backend with the AI/model inference pipeline.
- Handle file validation, storage, preprocessing flow, and result handling.
- Coordinate the work between backend, frontend, and research.
- Track what each team member is doing and make sure work is moving properly.
- Maintain the main project structure and ensure all parts connect smoothly.

### Key Deliverables

- Backend API structure.
- Upload and result endpoints.
- Processing/status flow.
- Integration with model pipeline.
- Final coordination of frontend-backend connection.

---

## 2. Venkyy — Research Lead

Venkyy will focus on understanding the research side and explaining the project clearly to the team.

### Main Responsibilities

- Research LISS-IV satellite imagery and the cloud-removal problem.
- Understand existing AI approaches such as GANs, diffusion models, and multimodal fusion.
- Explain the project concepts to the team in simple language.
- Help the team understand why each technical choice is being made.
- Support presentation content, methodology explanation, and project documentation.
- Compare reference solutions and identify useful ideas.

### Key Deliverables

- Clear explanation of the problem statement.
- Research notes on models and datasets.
- Support for methodology and PPT content.
- Simple explanations for judges and teammates.

---

## 3. Laksha — Frontend Developer

Laksha will work on building the frontend interface of the project.

### Main Responsibilities

- Develop frontend pages using React, Vite, and Tailwind CSS.
- Work on the Home Page, Upload Page, and Methodology Page.
- Build clean UI components such as cards, buttons, upload sections, and layout blocks.
- Make the app responsive for different screen sizes.
- Coordinate with Rockie to connect frontend forms with backend APIs.

### Key Deliverables

- Home Page.
- Upload Page.
- Methodology Page.
- Reusable UI components.
- Responsive page layouts.

---

## 4. Mru — Frontend Developer

Mru will work on the frontend result experience and visual polish.

### Main Responsibilities

- Develop the Processing Page and Results Page.
- Create animations, progress indicators, and loading states.
- Build the before/after image comparison section.
- Work on metrics display cards and result download section.
- Improve the visual style with satellite-themed UI, transitions, and polish.
- Coordinate with Laksha to keep frontend styling consistent.

### Key Deliverables

- Processing Page.
- Results Page.
- Before/after viewer.
- Metrics panel.
- Animations and UI polish.

---

## Team Workflow

1. Venkyy clarifies the concept, model approach, and project explanation.
2. Rockie sets up backend flow and coordinates tasks.
3. Laksha and Mru build frontend pages and components.
4. Rockie connects frontend with backend APIs.
5. Everyone reviews the final demo flow together.
6. Venkyy helps prepare the final explanation and presentation.

---

## Final Responsibility Summary

- **Rockie:** Backend + project management + integration.
- **Venkyy:** Research + explanation + documentation/presentation support.
- **Laksha:** Frontend structure + main pages + UI components.
- **Mru:** Frontend results flow + animations + visual polish.
